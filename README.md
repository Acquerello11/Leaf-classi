# 🌿 Leaf Classification AI (ระบบจำแนกสายพันธุ์ใบไม้ด้วย AI)

โปรเจกต์นี้คือระบบปัญญาประดิษฐ์สำหรับคัดแยกและจำแนกสายพันธุ์ของใบไม้ โดยใช้โมเดล **DINOv2** (จาก Facebook Research) ในการสกัดจุดเด่นของภาพ (Feature Extraction) และใช้ **Support Vector Machine (SVM)** ในการจำแนกคลาส (Classification) ระบบมาพร้อมกับ Web UI ที่สร้างด้วย **Gradio** เพื่อให้ง่ายต่อการใช้งานและทดสอบความแม่นยำของโมเดล

## ⚙️ Pipeline ขั้นตอนการสร้างโมเดลโดยละเอียด

กระบวนการสร้าง AI ของโปรเจกต์นี้ ถูกออกแบบมาอย่างเป็นระบบเพื่อให้ได้โมเดลที่มีความแม่นยำสูงสุด โดยแบ่งออกเป็น 2 ขั้นตอนหลัก (2 สคริปต์) ดังนี้:

### 1. การทำความสะอาดและเตรียมข้อมูล (`clean_and_prepare_data.py`)
ขั้นตอนนี้คือการเตรียม "วัตถุดิบ" ให้ดีที่สุดก่อนส่งให้ AI เรียนรู้ (ลดปัญหา Garbage In, Garbage Out)
**เครื่องมือและเทคนิคที่ใช้:**
- **PyTorch & Pillow (PIL)**: ใช้โหลดรูปภาพต้นฉบับ ปรับขนาดภาพ (Resize) เป็น 448x448 พิกเซล และแปลงภาพเป็นตัวเลข (Tensor/Normalize)
- **DINOv2 (Vision Transformer)**: โหลดโมเดลจาก `torch.hub` มาทำหน้าที่เป็นดวงตาชั่วคราว เพื่อสแกนดูรูปภาพทั้งหมดในแต่ละสายพันธุ์ และแปลงรูปภาพเป็นเวกเตอร์
- **Scikit-Learn (Cosine Similarity)**: เมื่อแปลงรูปเป็นเวกเตอร์แล้ว จะใช้คณิตศาสตร์คำนวณหา "ค่าความเหมือน" (Cosine) ของรูปภาพทุกรูปเทียบกับค่ากลาง (Median) ประจำสายพันธุ์นั้นๆ
- **สถิติ Robust Z-Score (MAD)**: นำค่าความเหมือนมาเข้าสูตรสถิติ ถ้ารูปไหนคะแนนทะลุเกณฑ์ (Drop Threshold) แสดงว่าเป็น "ภาพขยะ" หรือ "ภาพแปลกปลอม" ระบบจะทำการ **ลบทิ้ง (Outlier Rejection)** ทันที
- **Shutil & Random**: เมื่อคัดภาพขยะทิ้งหมดแล้ว จะใช้การสุ่มสลับรูป และคัดลอกไฟล์ที่สะอาดแล้ว (Cleaned Data) แบ่งออกเป็นโฟลเดอร์ `train` (70%), `validation` (15%), `test` (15%) เพื่อเตรียมนำไปเทรนต่อไป

### 2. การฝึกสอนและสร้างสมอง AI (`Modeltrainer.py`)
เมื่อเราได้ข้อมูลรูปใบไม้ที่ "สะอาดหมดจด" (ไม่มีรูปแปลกปลอม) มาจากสเต็ปแรกแล้ว สเต็ปต่อไปคือการ **สร้าง "สมอง"** เพื่อให้คอมพิวเตอร์เรียนรู้และจดจำลักษณะของสายพันธุ์ใบไม้ต่างๆ 

ถ้าเปรียบเทียบให้เห็นภาพ ขั้นตอนนี้เหมือนเรากำลัง **ส่งนักเรียนเข้าไปฝึกวิชาในห้องสมุด**:

#### 🧠 2.1 สกัดจุดเด่นรูปภาพด้วย DINOv2
- **มันคืออะไร?**: เราใช้คำสั่งใน `PyTorch` โหลดโมเดล DINOv2 ขึ้นมา ซึ่งเปรียบเสมือน **"แว่นตาวิเศษสำหรับอ่านหนังสือ"** 
- **ช่วยทำอะไรได้?**: ปกติคอมพิวเตอร์จะไม่รู้จักคำว่า "สีเขียว" หรือ "ขอบใบหยัก" แว่นตา DINOv2 จะทำหน้าที่สแกนรูปใบไม้ทีละรูป แล้วสกัดความหมายเชิงลึกออกมาเป็น **"บาร์โค้ดประจำตัว"** (เวกเตอร์ตัวเลข 384 ตัว) 
- **ประโยชน์**: เราไม่ต้องเสียเวลาเขียนโปรแกรมบอกคอมพิวเตอร์ว่าใบไม้หน้าตาเป็นยังไง เพราะ DINOv2 จะช่วยดึง "จุดเด่น" ออกมาให้อัตโนมัติและแม่นยำมาก แม้รูปจะเบลอ หรือมีแสงเงาบังก็ตาม

#### ⚖️ 2.2 จำแนกสายพันธุ์ด้วย Support Vector Machine (SVM)
- **มันคืออะไร?**: เราใช้โมเดล SVM จาก `Scikit-Learn` ซึ่งเปรียบเสมือน **"กรรมการผู้ตัดสิน"** ที่คอยรับบาร์โค้ดใบไม้จาก DINOv2 มาวิเคราะห์
- **ช่วยทำอะไรได้?**: กรรมการ SVM จะเรียนรู้ว่า บาร์โค้ดหน้าตาแบบไหนคือสายพันธุ์ A และแบบไหนคือสายพันธุ์ B หน้าที่ของมันคือพยายาม **"ขีดเส้นแบ่งอาณาเขต"** ระหว่างสายพันธุ์ต่างๆ ออกจากกันให้ชัดเจนที่สุด
  - **ทำไมต้องขีดเส้นตรง (Linear Kernel)?**: เพราะบาร์โค้ดที่แว่นตา DINOv2 สกัดมาให้มันจับกลุ่มเป็นระเบียบมาก จน SVM ไม่ต้องใช้คณิตศาสตร์โค้งงอให้ปวดหัว แค่ขีดเส้นตรง (Linear) ก็แบ่งสายพันธุ์ได้ขาดแล้ว! **ประโยชน์คือ: เทรนโมเดลเสร็จไวมากๆ กินสเปคน้อย และทำนายผลรวดเร็ว**
  - **แก้ปัญหาข้อมูลไม่สมดุล (class_weight='balanced')**: สมมติใบกะเพรามี 1,000 รูป แต่มะกรูดมีแค่ 50 รูป ถ้าปล่อยไปตามปกติ กรรมการจะลำเอียงและทายแต่กะเพรา เราเลยใช้คำสั่ง 'balanced' ซึ่งเปรียบเหมือนการ **"แจกโทรโข่ง"** ให้กลุ่มมะกรูดที่มีรูปน้อย เพื่อให้มีน้ำหนักคะแนนเท่าเทียมกัน **ประโยชน์คือ: AI จะมีความยุติธรรม ไม่ทอดทิ้งสายพันธุ์หายาก**

#### 📦 2.3 บันทึกสมอง AI เพื่อใช้งานจริง (Pickle)
- **มันคืออะไร?**: เป็นการแพ็กข้อมูลทั้งหมดด้วยไลบรารี `Pickle` เปรียบเหมือนการ **"เซฟเกม"** 
- **ช่วยทำอะไรได้?**: เมื่อ SVM ขีดเส้นจำแนกสายพันธุ์เสร็จหมดแล้ว เราไม่จำเป็นต้องให้มันมานั่งเรียนรู้ขีดเส้นใหม่ทุกครั้งที่เปิดคอมพิวเตอร์ ระบบจะทำการเซฟความจำทั้งหมดแพ็กใส่กล่องเป็นไฟล์ชื่อ `dino_centroids_cleaned.pkl` 
- **ประโยชน์**: เราสามารถก็อปปี้ไฟล์นี้ไปเสียบใช้งานบนหน้าเว็บ (Web UI) หรือเอาไปรันบนคอมพิวเตอร์เครื่องอื่นเพื่อทำนายผลได้ทันที โดยไม่ต้องเสียเวลาเทรนใหม่ตั้งแต่ศูนย์

## 📂 ไฟล์อื่นๆ ในโปรเจกต์
- **`app.py`**: สคริปต์สร้าง Web UI ด้วย Gradio ให้ผู้ใช้อัปโหลดรูปมาทำนายได้
- **`evaluate_validation.py`**: สคริปต์ทดสอบหาความแม่นยำ (Accuracy) กับข้อมูลชุด Validation ที่โมเดลไม่เคยเห็น พร้อมวาด Confusion Matrix

## 🚀 วิธีการใช้งาน

### 1. การติดตั้งไลบรารี (Installation)
ก่อนเริ่มต้นใช้งาน ให้ติดตั้งไลบรารีที่จำเป็นทั้งหมดผ่านคำสั่ง:
```bash
pip install -r requirements.txt
```

### 2. การเตรียมข้อมูล (Data Preparation)
นำข้อมูลรูปภาพดิบไปวางไว้ที่ `T-Leaf(From3273080)/Data200` (หรือแก้ Path ในโค้ด) จากนั้นรันคำสั่ง:
```bash
python clean_and_prepare_data.py
```
*สคริปต์จะคัดรูปภาพที่ไม่ดีทิ้ง และสร้างโฟลเดอร์ `processed_data/data_cleaned` พร้อมแบ่งรูปภาพเป็น Train/Val/Test ให้โดยอัตโนมัติ*

### 3. การฝึกสอนโมเดล (Training the Model)
เมื่อเตรียมข้อมูลเสร็จแล้ว ให้รันคำสั่งเพื่อเริ่มกระบวนการฝึกสอนโมเดล:
```bash
python Modeltrainer.py
```

**สิ่งที่เกิดขึ้นในขั้นตอนนี้:**
1. **โหลด Backbone Model**: ระบบจะโหลดโมเดล `DINOv2` (`dinov2_vits14`) เพื่อทำหน้าที่เป็นตัวสกัดจุดเด่น (Feature Extractor) ที่ทรงพลังและมีความทนทาน (Robust)
2. **สกัด Features**: ระบบจะอ่านรูปภาพทั้งหมดจากโฟลเดอร์ `processed_data/data_cleaned/train` ส่งผ่านโมเดล DINOv2 เพื่อแปลงภาพให้เป็นเวกเตอร์ตัวเลข (Embeddings)
3. **ฝึกสอน SVM**: นำเวกเตอร์ที่สกัดได้ไปฝึกสอนโมเดล `Support Vector Machine (SVM)` ที่ตั้งค่าเป็น Linear Kernel พร้อมเปิดใช้งาน Probability และปรับน้ำหนักคลาสให้สมดุล (`class_weight='balanced'`) เพื่อรองรับกรณีที่จำนวนรูปภาพแต่ละสายพันธุ์ไม่เท่ากัน
4. **บันทึกโมเดล**: เมื่อเทรนเสร็จสิ้น ระบบจะแพ็กโมเดล SVM ลงในไฟล์ `dino_centroids_cleaned.pkl` เพื่อเตรียมไว้ใช้งานในส่วนของ Web UI

*เคล็ดลับ: หากมีข้อมูลสายพันธุ์ใหม่เพิ่มเติม สามารถนำรูปไปใส่ในโฟลเดอร์ Train แล้วรันสคริปต์นี้ใหม่เพื่ออัปเดตความรู้ให้ AI ได้เลย*

### 4. การรันระบบ Web UI (Inference & Web App)
เริ่มใช้งานระบบผ่านหน้าเว็บด้วยคำสั่ง:
```bash
python app.py
```
*ระบบจะแสดง URL (เช่น `http://127.0.0.1:7860`) ให้คลิกเพื่อเปิดใช้งานบนเบราว์เซอร์*

## 🛠️ ข้อมูลเชิงลึกและเทคโนโลยีที่ใช้ (Technical Info)

โปรเจกต์นี้เป็นการผสมผสานระหว่างเทคโนโลยี AI ขั้นสูงกับการออกแบบระบบที่เรียบง่าย เพื่อให้ได้โมเดลที่มีความแม่นยำและทนทานต่อข้อมูลที่หลากหลาย โดยมีส่วนประกอบหลักดังนี้:

### 1. DINOv2 และ Vision Transformer (เปรียบเทียบให้เห็นภาพ)
- **เครื่องมือที่ใช้**: `torch.hub.load('facebookresearch/dinov2', 'dinov2_vits14')`
- **Vision Transformer คืออะไร?**: ลองจินตนาการว่าเวลาคนเรามองรูปภาพ เราไม่ได้มองทีละพิกเซล แต่มองเป็น "ส่วนประกอบ" (เช่น ขอบใบไม้ เส้นเลือดใบ สี) Vision Transformer ทำงานคล้ายกัน โดยมันจะ **สับรูปภาพออกเป็นชิ้นเล็กๆ เหมือนตัวต่อจิ๊กซอว์ (Patches)** จากนั้นมันจะดูว่าชิ้นส่วนแต่ละชิ้นมีความสัมพันธ์กันอย่างไร (เช่น ชิ้นที่เป็นขอบใบไม้มักจะอยู่ติดกับชิ้นที่เป็นพื้นหลัง) ทำให้มันเข้าใจ "ภาพรวม" ของรูปได้ลึกซึ้งมาก
- **DINOv2 เจ๋งยังไง?**: โมเดล AI ทั่วไปต้องมีคนคอยป้อนรูปและบอกว่า "นี่คือใบไม้ A นะ" "นี่คือใบไม้ B นะ" นับแสนๆ รูป (Supervised Learning) แต่ DINOv2 (สร้างโดยทีม AI ของ Facebook/Meta) ถูกฝึกมาแบบ **เรียนรู้ด้วยตัวเอง (Self-Supervised Learning)** เหมือนปล่อยเด็กเข้าไปในป่าให้สังเกตต้นไม้เอง เด็กจะเริ่มรู้เองว่า "ใบแบบนี้รูปทรงคล้ายกัน" หรือ "สีแบบนี้มักมาด้วยกัน" ทำให้ DINOv2 เป็นเหมือน **"ผู้เชี่ยวชาญด้านการมองภาพ"** ที่มองเห็นลวดลายที่ซ่อนอยู่ (Features) ได้เก่งกว่าโมเดลทั่วไป แม้ภาพจะเบลอ หรือมีแสงเงาบดบัง
- **ใช้ทำอะไรในโปรเจกต์นี้?**: ทำหน้าที่เป็น **"ดวงตาอัจฉริยะ"** ที่จะแปลงรูปภาพใบไม้ให้กลายเป็น **ชุดตัวเลข 384 ตัว (Embeddings)** ซึ่งเปรียบเสมือน "บาร์โค้ด" ประจำตัวของใบไม้นั้นๆ 
- **ใช้ในส่วนไหน**: 
  - `clean_and_prepare_data.py`: สร้างบาร์โค้ดเพื่อหาว่ารูปไหนหน้าตาแปลกเพื่อน (ภาพขยะ) จะได้คัดทิ้ง
  - `Modeltrainer.py`: สร้างบาร์โค้ดจากรูปใบไม้ทั้งหมด เพื่อส่งให้สมอง (SVM) จดจำว่าบาร์โค้ดแบบนี้คือสายพันธุ์อะไร
  - `app.py`: สร้างบาร์โค้ดจากรูปที่ผู้ใช้อัปโหลด เพื่อให้สมอง (SVM) ทำนาย

### 2. Support Vector Machine (SVM) - สมองกลตัดสินใจ
- **เครื่องมือที่ใช้**: `sklearn.svm.SVC` (Scikit-learn)
- **SVM ทำงานยังไง?**: ลองจินตนาการว่าคุณมีลูกแก้วสีแดงและสีน้ำเงินวางปนกันอยู่บนโต๊ะ หน้าที่ของ SVM คือการหา **"ไม้บรรทัดมาวางกั้น"** ตรงกลาง เพื่อแยกพื้นที่ของลูกแก้วสีแดงและสีน้ำเงินออกจากกันให้ชัดเจนที่สุด ในโปรเจกต์นี้ ลูกแก้วก็คือ "บาร์โค้ดตัวเลข 384 ตัว" ที่ได้มาจากดวงตา (DINOv2) นั่นเอง
- **ทำไมถึงใช้ Linear Kernel?**: เพราะ DINOv2 สกัดบาร์โค้ดออกมาได้เป็นระเบียบและชัดเจนมาก จนเราสามารถใช้แค่เส้นตรงธรรมดา (Linear) ขีดแบ่งสายพันธุ์ได้เลย ไม่ต้องใช้สมการโค้งงอซับซ้อน ทำให้โปรแกรมทำงานเบา หวิว และเร็วมาก
- **เทคนิค Balanced Weight**: บางสายพันธุ์อาจมีรูปเยอะ บางสายพันธุ์มีรูปน้อย SVM เลยอาจจะแอบ "ลำเอียง" ไปทายสายพันธุ์ที่มีรูปเยอะบ่อยกว่า เราเลยสั่งให้ระบบปรับน้ำหนัก (`class_weight='balanced'`) เพื่อให้คะแนนความสำคัญกับสายพันธุ์รูปน้อยมากขึ้น ทำให้มันยุติธรรม
- **ใช้ในส่วนไหน**: `Modeltrainer.py` (ตอนเรียนรู้ที่จะขีดเส้นแบ่ง) และ `app.py` (ตอนตัดสินว่ารูปใหม่อยู่ฝั่งไหนของเส้น)

### 3. Outlier Rejection (ระบบเด้งรูปขยะทิ้งอัตโนมัติ)
- **เครื่องมือที่ใช้**: โคไซน์ (Cosine Similarity) และสถิติหาค่าผิดปกติ (Robust Z-Score)
- **ทำงานยังไงให้เห็นภาพ**: สมมติในกลุ่มมีคน 10 คนกำลังใส่เสื้อสีเขียวเหมือนกันหมด (ใบไม้สายพันธุ์เดียวกัน) แต่อยู่ดีๆ มีคนนึงใส่เสื้อสีแดงเดินเข้ามาปน (เช่น รูปพื้นหลังว่างๆ หรือรูปคน) 
  1. โปรแกรมจะหา **"หน้าตามาตรฐาน" (Median)** ของคนส่วนใหญ่ในกลุ่มนี้ก่อน
  2. จากนั้นจะเอาบาร์โค้ดของทุกคนมาเทียบกับหน้าตามาตรฐาน ว่าใครมีความคล้ายคลึงแค่ไหน (วัดด้วยมุม Cosine)
  3. ถ้าเจอรูปไหนที่คะแนนฉีกออกไปจากกลุ่มแบบสุดโต่ง (Z-Score ทะลุ Threshold) ระบบจะเตะรูปนั้นทิ้งลงถังขยะทันที เพื่อไม่ให้มันไปทำให้สมอง (SVM) สับสนตอนเทรน
- **ใช้ในส่วนไหน**: `clean_and_prepare_data.py`
