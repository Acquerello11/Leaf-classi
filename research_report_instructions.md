# รายงานการวิจัยสถาปัตยกรรม Hierarchical Classification / Mixture of Experts (MoE) สำหรับการคัดแยกสายพันธุ์พืช

## ทฤษฎีเบื้องหลังที่สำคัญ (Core Theoretical Background)

### หลักการ MAE (Masked Autoencoders) และ DINOv2
**MAE** เป็นสถาปัตยกรรมและกลไกการเรียนรู้เชิงลึกแบบ Self-Supervised Learning ซึ่งเป็นรากฐานของโมเดลตระกูล Vision Transformer อย่าง **DINOv2** ที่ใช้ในโปรเจกต์นี้ โดยมีหลักการทำงานดังนี้:
1. **Masking (การบดบังข้อมูล):** รูปภาพต้นฉบับจะถูกหั่นเป็นชิ้นเล็กๆ (Patches) จากนั้นระบบจะสุ่ม "ปกปิด (Mask)" ชิ้นส่วนเหล่านั้นทิ้งไปในสัดส่วนที่สูงมาก
2. **Reconstruction (การประกอบร่างคืน):** บังคับให้ AI พยายามพยากรณ์ชิ้นส่วนที่หายไปกลับคืนมาให้เหมือนรูปต้นฉบับที่สุด
3. **การประยุกต์ใช้ในระบบ (Impact):** การเรียนรู้ด้วยการทายภาพซ้ำๆ ทำให้ AI เกิดความเข้าใจใน "โครงสร้างและบริบทเชิงลึก (Deep Semantic Structure)" โดยไม่ต้องมีป้ายกำกับ (Label) สิ่งนี้ทำให้ DINOv2 สามารถสกัดเวกเตอร์เพื่อแยกภาพขยะ (`1clean_and_prepare_data.py`) และจัดกลุ่มอัตโนมัติ (`2split.py`) ได้อย่างแม่นยำในพื้นที่ของมิติเวกเตอร์ (Embedding Space)

## 1. การทำความสะอาดและจัดการข้อมูลคุณภาพต่ำ (`1clean_and_prepare_data.py`)

**หลักการ (Principle):**
เพื่อแก้ปัญหา "Garbage In, Garbage Out" ข้อมูลรูปภาพจำเป็นต้องถูกคัดกรองภาพขยะ (Outlier) ออกก่อนนำไปฝึกสอน โดยใช้การวิเคราะห์ความผิดปกติในพื้นที่เวกเตอร์

**กลไกการทำงาน (Mechanism):**
1. **Feature Extraction:** ใช้โมเดล **DINOv2** (`dinov2_vits14`) แปลงรูปภาพ (หลังถูกปรับขนาดเป็น 448x448 พิกเซล) ให้เป็นเวกเตอร์
2. **Prototype Calculation:** คำนวณค่ามัธยฐาน (Median) ของเวกเตอร์ภาพทั้งหมดในคลาสนั้น เพื่อใช้เป็นตัวแทนอุดมคติ (Prototype) ของสายพันธุ์
3. **Quality Sorting & Hard Test Split:** นำเวกเตอร์ของแต่ละภาพมาหาความคล้ายคลึง (Cosine Similarity) กับ Prototype เพื่อใช้เป็น "คะแนนความสมบูรณ์ของใบไม้" โดยไม่มีการคัดภาพทิ้งเลยแม้แต่รูปเดียว (No Data Drop) ภาพที่คะแนนสูง (70%) จะถูกดึงเข้า Train ภาพระดับกลาง (15%) เข้า Validation และภาพหางแถวที่คาดว่าเป็นภาพแปลกปลอมหรือมี Noise มาก (15%) จะถูกดันเข้าชุด Test ทั้งหมด เพื่อใช้เป็น Hard Example ทดสอบความทนทานของโมเดลในการใช้งานจริง
4. **Data Splitting & Resume:** ระบบจัดเตรียมโฟลเดอร์ให้อัตโนมัติ พร้อมระบบ Resume ที่ช่วยข้ามคลาสที่ประมวลผลเสร็จแล้ว ช่วยให้ทำงานได้อย่างต่อเนื่องกรณีที่หยุดกลางคัน

## 2. การจัดหมวดหมู่สายพันธุ์แบบอัตโนมัติ (`2split.py`)

**หลักการ (Principle):**
การจำแนกคลาสที่มีจำนวนมาก (เช่น 61 สายพันธุ์) ด้วยโมเดลเดียวมักทำให้เกิดคอขวดของประสิทธิภาพ ระบบจึงแก้ไขโดยใช้เทคนิค Unsupervised Clustering เพื่อจับกลุ่มพืชที่มีลักษณะทางกายภาพคล้ายคลึงกัน (Visual Similarity) ให้อยู่ใน "กลุ่มใหญ่" (Super Groups) เดียวกันก่อน

**กลไกการทำงาน (Mechanism):**
1. **High-Speed Embedding:** ใช้ DINOv2 แปลงภาพขนาด 224x224 พิกเซลจากทุกคลาสเป็นเวกเตอร์ด้วยระบบ Batch Processing (`DataLoader` `batch_size=128`) เพื่อความเร็วสูงสุด 
2. **Auto K-Means Clustering:** นำเวกเตอร์มัธยฐานประจำคลาส มาหาจำนวนกลุ่มที่เหมาะสมที่สุดผ่านอัลกอริทึม **K-Means** โดยให้ AI ค้นหาจำนวนกลุ่มที่แบ่งได้ละเอียดที่สุดในช่วง 12 ถึง 25 กลุ่ม 
3. **Silhouette Optimization:** ระบบจะประเมินความชัดเจนของกลุ่มด้วยคะแนน **Silhouette Score** โดยค่า K ที่ให้คะแนนสูงสุดจะถูกเลือกใช้เป็นจำนวนกลุ่มที่เหมาะสมที่สุด
4. **Hierarchical Directory Creation:** ย้ายข้อมูลสายพันธุ์ย่อยทั้งหมด จัดเก็บเข้าสู่โฟลเดอร์กลุ่มใหญ่ตามที่ K-Means ทำการแบ่งไว้

## 3. การสร้างโมเดลจำแนกกลุ่มหลัก หรือ นายประตู (`3train.py`)

**หลักการ (Principle):**
โมเดลกลุ่มหลัก (Master Router Model) ทำหน้าที่เหมือน "นายประตู" คัดกรองภาพว่าควรถูกส่งไปให้ผู้เชี่ยวชาญกลุ่มใด (Super Group) วิเคราะห์ต่อ โดยใช้แนวทาง **Linear Probing** เพื่อให้โมเดลมีขนาดเล็ก ฝึกสอนเร็ว และลดการเกิด Overfitting

**กลไกการทำงาน (Mechanism):**
1. **Dataset Loading:** สร้าง `SuperGroupDataset` ที่ดึง Label ออกมาเป็น ID ของกลุ่มใหญ่ (Super Group) แทนการใช้สายพันธุ์ย่อย
2. **Frozen Backbone + Linear Probe:** ใช้ DINOv2 เป็นฐานข้อมูลแบบแช่แข็งน้ำหนัก (ไม่ต้องอัปเดต Gradient) สำหรับสกัดเวกเตอร์ความยาว 384 มิติ แล้วต่อด้วย Linear Layer ตัวเล็ก (`DINORouter`) 1 ชั้นเพื่อพยากรณ์กลุ่ม
3. **Optimized DataLoader:** ออกแบบคิวการป้อนข้อมูลประสิทธิภาพสูง ใช้ `num_workers=8`, `pin_memory=True`, `persistent_workers=True` ควบคู่กับการใช้การ์ดจออย่างเต็มที่

## 4. การฝึกสอนโมเดลผู้เชี่ยวชาญเฉพาะกลุ่ม (`4trainexpert.py`)

**หลักการ (Principle):**
ผู้เชี่ยวชาญ (Expert Model) จะฝึกสอนเฉพาะในกลุ่มพืชที่มีลักษณะหน้าตาคล้ายกันมากๆ โดยประยุกต์ใช้เทคนิค **Deep Metric Learning**, **High-Resolution Pipeline**, และ **Hard Example Mining** 

**กลไกการทำงาน (Mechanism):**
1. **High-Resolution Pipeline:** เพิ่มขนาดภาพที่ใช้ Train และ Validate ให้สูงระดับ **448x448 พิกเซล** ข้อมูลจะถูกดึงเข้า `ResNet50` เพื่อเก็บรักษารายละเอียดไมโครฟีเจอร์ เช่น รูปแบบเส้นใบ (Venation) 
2. **ArcFace Loss Integration:** ใช้เลเยอร์คณิตศาสตร์ `ArcMarginProduct` (สวมทับที่ส่วนหัวของโมเดลแทน Linear ธรรมดา) เพื่อทำหน้าที่บีบให้ภาพในคลาสเดียวกันมีความแคบลง (Compactness) และขยาย Margin ผลักคลาสอื่นๆ ให้ห่างออกจากกันเมื่อวัดด้วย Cosine Distance
3. **Hard Negative Mining:** ใช้ฟังก์ชัน **Focal Loss** (ด้วยค่า $\gamma = 2.0$) แทนที่ Cross Entropy ซึ่งจะไปลดน้ำหนักการเรียนรู้ของภาพที่ตอบถูกแล้ว และหันไปบีบเค้นทำโทษให้โมเดลเรียนรู้หนักๆ กับภาพลักษณะ "คู่เหมือน" หรือภาพที่โมเดลยังสับสน

## 5. เครื่องยนต์ประมวลผลและการควบรวมโมเดล (`5mixmodel.py`)

**หลักการ (Principle):**
Inference Engine ออกแบบกลไกเลียนแบบระบบ Top-down Processing (มองภาพรวมก่อน แล้วเจาะหารายละเอียด) รองรับท่อข้อมูลแบบหลายความละเอียดภาพ (Dual-Resolution) ควบรวมความน่าจะเป็นด้วยกฎทฤษฎีความน่าจะเป็น (Ensemble Probability)

**กลไกการทำงาน (Mechanism):**
1. **Dual-Resolution Transform:** แบ่งท่อแปลงขนาดภาพ 2 ระดับ: ภาพขนาด 224px ถูกส่งให้นายประตู (DINOv2) และ 448px ส่งให้ผู้เชี่ยวชาญ (ResNet50)
2. **Master Routing (ด่านที่ 1):** ภาพส่งเข้า DINOv2 Router เพื่อพยากรณ์ความน่าจะเป็นระดับกลุ่มหลัก (Group Probability) ด้วย Softmax
3. **Expert Activation (ด่านที่ 2):** เมื่อทราบว่าโมเดลควรส่งภาพไปให้กลุ่มใด, เครื่องยนต์จะโหลด `ExpertNet` ประจำกลุ่มนั้น มาคำนวณระยะห่าง (Cosine Distance ผ่าน ArcFace Weight) แปลงกลับมาเป็นระดับสายพันธุ์ (Species Probability)
4. **Ensemble Probability Computation:** คำนวณความน่าจะเป็นสุทธิด้วยสมการ $P(Species) = P(Group) \times P(Species | Group)$ เพื่อจัดอันดับความเป็นไปได้ของสายพันธุ์ที่แม่นยำที่สุด (Top-K)

## 6. การประเมินประสิทธิภาพของสถาปัตยกรรมทั้งหมด (`6evaluate.py`)

**หลักการ (Principle):**
ทดสอบประสิทธิผลของระบบ End-to-End ตั้งแต่ต้นน้ำจนถึงปลายน้ำ เพื่อพิสูจน์การทำงานที่แท้จริงบนชุดข้อมูลทดสอบ (Test Set) เชิงสถิติ

**กลไกการทำงาน (Mechanism):**
1. **Pipeline Execution:** ดึงชุดข้อมูลที่ถูกเก็บไว้ใน Test Directory ทุกภาพเข้ามาประมวลผลผ่าน `PlantInferenceEngine` เพื่อจำลองการใช้งานจริง
2. **Metric Calculation & Automated Reporting:** วัดและรวบรวมค่าทางสถิติที่สำคัญ ได้แก่ Overall Accuracy พร้อมแจกแจง Classification Report สรุปค่า Precision, Recall, และ F1-Score ประจำแต่ละคลาสอย่างละเอียด โดยส่งออกเป็นไฟล์ `evaluation_report.csv` เพื่อให้นำไปทำตารางรายงานได้ทันที
3. **Confusion Matrix & Training Curves:** พล็อตกราฟ **Confusion Matrix** (Heatmap) ที่รวมทั้ง 61 คลาสเข้าไว้ด้วยกัน และกราฟแสดงผลการเทรน (Loss & Accuracy) ของทุกโมเดล (ไฟล์ 3 และไฟล์ 4)
4. **Explainable AI (Grad-CAM):** สร้างภาพ Heatmap จากโมเดล Expert (ไฟล์ 4) เพื่อแสดงให้เห็นว่าบริเวณใดของใบไม้ (เช่น เส้นใบ, ขอบใบ) ที่ AI ให้ความสำคัญในการตัดสินใจสายพันธุ์

## ระบบอัตโนมัติแบบครบวงจร (Automated Pipeline)
เพื่อลดความผิดพลาดของมนุษย์ (Human Error) และเพิ่มความรวดเร็ว ระบบถูกออกแบบมาให้รันผ่าน `run_pipeline.py` เพียงคำสั่งเดียว โดยระบบจะทำการจัดการข้อมูล (ไฟล์ 1), แบ่งกลุ่ม K-Means (ไฟล์ 2), เทรน Master (ไฟล์ 3), วนลูปเทรน Expert ย่อยอัตโนมัติ (ไฟล์ 4), และสรุปผลประเมิน (ไฟล์ 6) อย่างต่อเนื่องจนจบกระบวนการ
