# คู่มือและหลักการทำงานของระบบ Hierarchical Classification สำหรับการคัดแยกสายพันธุ์พืช

เอกสารฉบับนี้จัดทำขึ้นเพื่อใช้เป็นข้อมูลอ้างอิงในการเขียนรายงานการวิจัย (Research Report) หรือโครงงาน โดยอธิบายหลักการทำงานเชิงลึก (Principles) และกลไกการทำงาน (Mechanisms) ของสคริปต์ทั้ง 6 ไฟล์ ซึ่งประกอบขึ้นเป็นสถาปัตยกรรมแบบ **Hierarchical / Mixture of Experts (MoE)** ระดับ State-of-the-Art

---

## ทฤษฎีเบื้องหลังที่สำคัญ (Core Theoretical Background)

### หลักการ MAE (Masked Autoencoders)
**MAE** เป็นสถาปัตยกรรมและกลไกการเรียนรู้เชิงลึกแบบ Self-Supervised Learning (เรียนรู้ด้วยตนเอง) ซึ่งเป็นรากฐานความสำเร็จของโมเดลตระกูล Vision Transformer รวมถึง **DINOv2** ที่ใช้ในโปรเจกต์นี้ โดยมีหลักการทำงานดังนี้:
1. **Masking (การบดบังข้อมูล):** ระบบจะนำรูปภาพต้นฉบับมาหั่นเป็นชิ้นเล็กๆ (Patches) เหมือนตัวต่อจิ๊กซอว์ จากนั้นจะสุ่ม "เซ็นเซอร์ หรือ ปกปิด (Mask)" ชิ้นส่วนเหล่านั้นทิ้งไปในสัดส่วนที่สูงมาก (มักจะสูงถึง 75% ของรูปทั้งหมด)
2. **Reconstruction (การประกอบร่างคืน):** บังคับให้ AI (Autoencoder) พยายามพยากรณ์และระบายสีชิ้นส่วนที่หายไปกลับคืนมาให้เหมือนรูปต้นฉบับที่สุด โดยอาศัยข้อมูลจาก 25% ที่เหลืออยู่
3. **การประยุกต์ใช้ในระบบ (Impact):** การถูกแกล้งให้ทายภาพที่หายไปซ้ำๆ กับรูปภาพนับล้านรูป ทำให้ AI เกิดความเข้าใจใน "โครงสร้างและบริบทเชิงลึก (Deep Semantic Structure)" โดยไม่ต้องมีมนุษย์คอยบอก (Label) ว่ารูปนี้คืออะไร 
   - ในโปรเจกต์นี้ กลไกที่คล้ายคลึงกับ MAE ทำให้ DINOv2 มีความฉลาดล้ำลึก ส่งผลให้สคริปต์ `1clean_and_prepare_data.py` สามารถสกัดเวกเตอร์เพื่อจับผิดภาพขยะได้อย่างแม่นยำ และช่วยให้การจัดกลุ่มอัตโนมัติใน `2split.py` แบ่งกลุ่มใบไม้ได้สมบูรณ์แบบแม้แสงและมุมกล้องจะเปลี่ยนไป

## 1. การทำความสะอาดและจัดการข้อมูลคุณภาพต่ำ (`1clean_and_prepare_data.py`)

**หลักการ (Principle):**
ในกระบวนการ Machine Learning ปัญหา "Garbage In, Garbage Out" เป็นสิ่งที่ส่งผลกระทบต่อความแม่นยำของโมเดลอย่างมาก สคริปต์นี้ใช้หลักการของการตรวจจับข้อมูลผิดปกติ (Anomaly/Outlier Detection) ในพื้นที่เวกเตอร์แรงดึงดูด (Embedding Space) โดยใช้ Vision Transformer ขั้นสูงเป็นตัววัดระยะห่าง

**การทำงาน (Mechanism):**
1. **Feature Extraction:** ใช้โมเดล **DINOv2 (Vision Transformer)** สกัดคุณลักษณะเด่นของรูปภาพทั้งหมดออกมาเป็นเวกเตอร์
2. **Prototype Calculation:** คำนวณหา "ค่ากลาง" (Median) ของเวกเตอร์ในแต่ละสายพันธุ์ เพื่อสร้างเป็นตัวแทนอุดมคติ (Prototype) ของสายพันธุ์นั้น
3. **Outlier Rejection:** คำนวณระยะห่างระหว่างรูปภาพแต่ละรูปกับค่ากลางด้วย **Cosine Similarity** หากรูปภาพใดมีระยะห่างเกินค่ามาตรฐานวิกฤต (Robust Z-Score / MAD Drop Threshold > 1.5) ระบบจะมองว่าเป็น "ภาพขยะ" หรือ "ภาพแปลกปลอม" และทำการตัดทิ้งอัตโนมัติ
4. **Data Splitting:** แบ่งข้อมูลที่ผ่านการทำความสะอาดแล้วออกเป็น 3 ชุดคือ Train (70%), Validation (15%), และ Test (15%) เพื่อป้องกันภาวะ Overfitting

## 2. การจัดหมวดหมู่สายพันธุ์แบบอัตโนมัติ (`2split.py`)

**หลักการ (Principle):**
การจำแนกคลาสที่มีจำนวนมาก (เช่น 61 สายพันธุ์) ด้วยโมเดลเดียวมักทำให้เกิดคอขวดของประสิทธิภาพ (Bottleneck) สคริปต์นี้แก้ปัญหาโดยใช้การจัดกลุ่มแบบไม่รู้จบ (Unsupervised Clustering) เพื่อรวมสายพันธุ์ที่มีลักษณะทางกายภาพคล้ายคลึงกัน (Visual Similarity) เข้าไว้ใน "กลุ่มใหญ่" (Super Groups) ก่อน

**การทำงาน (Mechanism):**
1. **Embedding Generation:** ใช้ DINOv2 สร้างเวกเตอร์ของแต่ละสายพันธุ์
2. **Auto K-Means Clustering:** นำเวกเตอร์เหล่านั้นมาเข้าอัลกอริทึม **K-Means** โดยระบบจะวนลูปทดสอบตั้งแต่ 3 ถึง 15 กลุ่ม เพื่อหาจำนวนกลุ่มที่เหมาะสมที่สุด
3. **Silhouette Optimization:** ใช้ค่า **Silhouette Score** เป็นเกณฑ์วัดว่าโมเดลควรจัดเป็นกี่กลุ่มใหญ่ (คะแนนยิ่งเข้าใกล้ 1 ยิ่งดี) ระบบจะเลือกจำนวนกลุ่ม K ที่ได้คะแนนสูงสุดอัตโนมัติ
4. **Hierarchical Directory Creation:** จัดโครงสร้างโฟลเดอร์ใหม่ โดยย้ายคลาสย่อยต่างๆ เข้าไปอยู่ภายใต้โฟลเดอร์กลุ่มใหญ่ (`super_group_X`)

## 3. การสร้างโมเดลจำแนกกลุ่มหลัก หรือ นายประตู (`3train.py`)

**หลักการ (Principle):**
โมเดลกลุ่มหลัก (Master Router Model หรือ Gate Network) ทำหน้าที่เสมือน "นายประตู" ที่จะคัดกรองว่าภาพใบไม้ที่รับเข้ามานั้น ควรถูกส่งไปให้ผู้เชี่ยวชาญกลุ่มใด (Super Group) วิเคราะห์ต่อ โดยสถาปัตยกรรมใหม่นี้จะเน้นการใช้เทคนิค **Linear Probing** บนเวกเตอร์ที่สกัดจากโมเดลระดับโลก เพื่อความรวดเร็วและใช้ทรัพยากรต่ำสุดขีด

**การทำงาน (Mechanism):**
1. **Dataset Loading:** สร้าง Custom Dataset (`SuperGroupDataset`) โดยใช้ชื่อของกลุ่มใหญ่ (Super Group) เป็น Label แทนที่จะเป็นชื่อสายพันธุ์ย่อย
2. **DINOv2 + Linear Probe Backbone:** โหลดโมเดล **DINOv2** เข้ามาในสถานะ "แช่แข็งน้ำหนัก (Frozen)" เพื่อทำหน้าที่สกัดเวกเตอร์ 384 มิติ จากนั้นต่อเข้ากับ **Linear Layer (DINORouter)** เล็กๆ 1 ชั้นเพื่อทำการพยากรณ์
3. **Training & Validation:** เนื่องจาก DINOv2 สามารถจัดเรียงข้อมูลแบบ Linearly Separable ได้ดีเยี่ยมอยู่แล้ว การเทรนจึงเกิดขึ้นเฉพาะที่ Linear Layer ทำให้การฝึกสอนเสร็จสิ้นไวมาก และป้องกันปัญหา Overfitting ได้เป็นอย่างดี

## 4. การฝึกสอนโมเดลผู้เชี่ยวชาญเฉพาะกลุ่ม (`4trainexpert.py`)

**หลักการ (Principle):**
โมเดลผู้เชี่ยวชาญ (Expert Model) จะโฟกัสการจำแนกพืชเพียงไม่กี่สายพันธุ์ที่มีหน้าตาคล้ายกันมากๆ ในสเต็ปนี้ได้นำเทคนิคระดับสูง 3 อย่างมาใช้คือ: **Deep Metric Learning**, **High-Resolution Pipeline**, และ **Hard Example Mining**

**การทำงาน (Mechanism):**
1. **High-Resolution Pipeline:** ขยายขนาดภาพ Input จาก 224px เป็น **448x448 พิกเซล** เพื่อรักษาความละเอียดของจุดต่างระดับไมโคร (Micro-features) เช่น ลักษณะขอบใบ (Margin) หรือเส้นใบย่อย (Venation) ไว้ให้ Convolutional Network (ResNet50) ประมวลผล
2. **ArcFace Loss Integration:** ใช้เทคนิค **ArcMarginProduct (ArcFace)** แทน Softmax เพื่อเพิ่มระยะห่างระหว่างคลาส (Margin) บังคับให้เวกเตอร์ของคลาสเดียวกันเกาะกลุ่มกันแน่นขึ้น และผลักคลาสที่ต่างกันให้ออกห่างกัน
3. **Hard Example Mining ด้วย Focal Loss:** นำสมการ **Focal Loss** เข้ามาแทน CrossEntropyLoss เพื่อดัดหลังโมเดล โดย Focal Loss จะปรับลดน้ำหนัก (Down-weight) ภาพที่โมเดลทายถูกได้ง่าย และทุ่มเท Gradient ไปที่ "ภาพคู่เหมือนที่โมเดลสับสนบ่อยๆ" เท่านั้น
4. **Specialized Training:** เทรนทีละกลุ่มอย่างอิสระ เมื่อเสร็จสิ้นจะเซฟไฟล์แยกรหัสกัน เช่น `expert_group_0.pth`

## 5. เครื่องยนต์ประมวลผลและการควบรวมโมเดล (`5mixmodel.py`)

**หลักการ (Principle):**
นี่คือแกนกลางของระบบอนุมาน (Inference Engine) ที่จำลองการทำงานของสมองมนุษย์แบบ Top-down Processing (มองภาพรวมก่อน แล้วค่อยเจาะรายละเอียด) โดยรองรับการอ่านภาพแบบ Dual-Resolution และควบรวมความน่าจะเป็นตามทฤษฎี Conditional Probability

**การทำงาน (Mechanism):**
1. **Dual-Resolution Transform:** ระบบถูกออกแบบให้รองรับการแปลงภาพ 2 สเกลพร้อมกัน คือ 224px สำหรับส่งให้นายประตู (DINOv2) และ 448px สำหรับส่งให้ผู้เชี่ยวชาญ (ResNet50)
2. **Master Routing (ด่านที่ 1):** ภาพขนาด 224px ถูกส่งเข้าโมเดล DINOv2 และ DINORouter เพื่อพยากรณ์ความน่าจะเป็นของ Super Group (Group Probability)
3. **Expert Activation (ด่านที่ 2):** ภาพขนาดความละเอียดสูง 448px ถูกส่งต่อไปยัง Expert Model ของกลุ่มที่ชนะในด่านแรก เพื่อพยากรณ์หาความน่าจะเป็นของสายพันธุ์ย่อย (Species Probability)
4. **Ensemble Probability Computation:** คำนวณความน่าจะเป็นสุทธิด้วยสมการ: 
   $P(Species) = P(Group) \times P(Species | Group)$
5. **Output Generation:** เรียงลำดับเปอร์เซ็นต์ความคล้ายคลึงสูงสุด (Top-K) และแสดงผลลัพธ์สุดท้าย

## 6. การประเมินประสิทธิภาพของสถาปัตยกรรมทั้งหมด (`6evaluate.py`)

**หลักการ (Principle):**
เพื่อพิสูจน์ว่าสถาปัตยกรรมทั้งหมด (DINOv2 Router + High-Res Expert) ทำงานได้จริงและไม่เกิด Data Leakage สคริปต์นี้ทำหน้าที่เป็นตัวชี้วัดประสิทธิภาพของระบบในเชิงสถิติ (Statistical Evaluation) โดยใช้ Test Set ล้วนๆ

**การทำงาน (Mechanism):**
1. **Pipeline Execution:** นำภาพจาก Test Set ทุกรูป ส่งเข้า Inference Engine (`5mixmodel.py`) ด้วยระบบการ import โค้ดที่เชื่อมต่อโมดูลได้อย่างอิสระ
2. **Metric Calculation:** วัดและรวบรวมค่าทางสถิติที่สำคัญ ทั้ง Overall Accuracy, Precision, Recall, และ F1-Score ผ่าน `classification_report` 
3. **Confusion Matrix Visualization:** สร้างเมทริกซ์ความสับสน (Confusion Matrix) ในรูปแบบ Heatmap เพื่อแสดงให้เห็นว่าระบบ Hard Example Mining ช่วยลดความสับสนระหว่างพืชสายพันธุ์คู่แฝดลงได้ดีเพียงใด
4. **Report Generation:** บันทึกกราฟเป็นไฟล์ `confusion_matrix_result.png` สำหรับนำไปใช้ตั้งเป็นผลการทดลองในงานวิจัย
